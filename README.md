# Full-Stack Interview Preparation

A structured, open collection of notes and Q&A for **full-stack engineering interviews** — from fundamentals through mid-level depth, senior architecture, system design, and framework internals.

Use it to prepare for interviews, mentor others, or keep a single place for concepts you revisit before design reviews.

## How to use this repo

1. **Pick your level** — Don’t read everything linearly; match the role you’re targeting.
2. **Senior interviews** — Prioritize trade-offs and real examples from your experience, not only definitions. The [Senior Level README](Senior%20Level%20Developer/README.md) explains this mindset.
3. **System design** — Clarify requirements, estimate capacity, then defend trade-offs and failure modes. See the [System Design Kit](System%20Design%20Interview%20Kit/README.md).
4. **Framework rounds** — Use [Senior Framework Questions](Senior%20Framework%20Questions/README.md) when interviewers go deep on runtime, APIs, and data layer.

## Topics covered (high level)

- **Architecture:** Scalability, monolith vs microservices, DDD, clean/hexagonal architecture, platform engineering  
- **Backend & scale:** High throughput, payments, idempotency, webhooks, gateways, queues, search  
- **Distributed systems:** Consensus, hashing, replication, resilience, chaos engineering  
- **Cloud & ops:** Observability (MELT), OpenTelemetry, security, AWS-style patterns  
- **Leadership:** Mentorship, reviews, incidents, estimation, technical debt, culture  
- **System design:** URL shorteners, chat, streaming, search, e-commerce, big data, security, real-world company architectures  
- **Frameworks:** V8/event loop, TypeScript, GraphQL, React/Next, PostgreSQL, IaC, stakeholder communication
